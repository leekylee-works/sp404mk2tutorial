# 📦 一鍵下載專案指南

## 🎯 目標
直接下載整個專案的 ZIP 檔案，不需要手動複製每個檔案

---

## 方法 1️⃣：使用 Figma Make 內建下載功能（最推薦）

### 在 Figma Make 界面中：

#### 選項 A：主選單下載
1. 查看左上角的 **主選單**（三條橫線 ≡ 或檔案選單）
2. 尋找 **「Download」** 或 **「Export」** 選項
3. 點擊後會下載整個專案的 ZIP 檔案
4. 解壓縮到本地資料夾

#### 選項 B：專案設定下載
1. 查看右上角的 **設定圖示** ⚙️
2. 尋找 **「Download Project」** 或 **「Export Project」**
3. 下載 ZIP 檔案
4. 解壓縮

#### 選項 C：快捷鍵
- 嘗試按 `Ctrl + Shift + E`（Windows）
- 或 `Cmd + Shift + E`（Mac）
- 可能會觸發下載

---

## 方法 2️⃣：使用 GitHub Desktop（如果下載不可用）

如果 Figma Make 沒有直接下載功能，這是最簡單的替代方案：

### 為什麼這個方法更好？

**傳統方法的問題：**
- ❌ 需要手動複製 18+ 個檔案
- ❌ 容易遺漏檔案
- ❌ 耗時 30-60 分鐘

**GitHub Desktop 的優勢：**
- ✅ Figma Make → GitHub 一鍵同步
- ✅ 自動處理所有檔案和資料夾
- ✅ 5-10 分鐘完成

### 具體步驟：

#### 1. 在 Figma Make 中連接 GitHub（如果尚未連接）

**查找同步或發布功能：**
1. 左上角選單 → 尋找 **「Publish to GitHub」** 或 **「Connect to GitHub」**
2. 右上角 → 尋找 **GitHub 圖示** 或 **「Deploy」** 按鈕
3. 點擊後授權連接到您的 GitHub 帳號
4. 選擇 Repository：`leekylee-works/sp404mk2tutorial`
5. 點擊 **「Push」** 或 **「Publish」**

**Figma Make 會自動：**
- ✅ 上傳所有 18 個核心檔案
- ✅ 保持正確的資料夾結構
- ✅ 處理所有依賴關係

#### 2. 完成！
- 前往 GitHub：https://github.com/leekylee-works/sp404mk2tutorial
- 確認所有檔案已上傳
- 前往 Vercel 部署

## 方法 4️⃣：直接在 GitHub 網頁上創建（不需下載）

這是最直接的方法，繞過下載步驟：

### 步驟：

1. **前往 GitHub Repository**
   - https://github.com/leekylee-works/sp404mk2tutorial

2. **清理舊檔案**
   - 刪除所有舊檔案

3. **在 Figma Make 和 GitHub 之間切換**
   - 開啟兩個瀏覽器視窗
   - 左側：Figma Make
   - 右側：GitHub

4. **逐個創建並複製**
   - Figma Make：點擊檔案 → Ctrl+A → Ctrl+C
   - GitHub：Create new file → 檔案名 → Ctrl+V → Commit

5. **使用清單確保不遺漏**
   - 使用 `📋_18個核心檔案清單.md`
   - 逐個打勾確認

**優點：**
- ✅ 不需要下載
- ✅ 不需要安裝軟體
- ✅ 直接上傳到 GitHub

**缺點：**
- ❌ 需要創建 18 次
- ❌ 約 30 分鐘

### 下載後的步驟：

#### 1️⃣ 解壓縮 ZIP 檔案
- 解壓到桌面或您選擇的位置
- 確認看到專案資料夾

#### 2️⃣ 檢查檔案結構
確認看到：
```
sp404mk2tutorial/
├── index.html
├── main.tsx
├── App.tsx
├── package.json
├── vite.config.ts
├── tsconfig.json
├── vercel.json
├── .gitignore

│   └── (6 個 .tsx 檔案)

│   └── (2 個 .ts 檔案)
├── types/
│   └── guide.ts
└── styles/
    └── globals.css
```

#### 3️⃣ 使用 GitHub Desktop 上傳

1. **開啟 GitHub Desktop**
2. **Clone Repository**
   ```
   leekylee-works/sp404mk2tutorial
   ```
3. **清理舊檔案**
   - 刪除 Clone 資料夾中的所有內容（保留 .git）
4. **複製解壓縮的檔案**
   - 將下載的專案檔案全部複製到 Clone 的資料夾
5. **Commit 和 Push**
   ```
   Message: Deploy complete version
   ```

#### 4️⃣ 前往 Vercel 部署
- https://vercel.com/
- Import Project
- 選擇 `sp404mk2tutorial`
- Root Directory: `./`
- Deploy
